# Змінні

Змінні і операції над ними — це одна з найпростіших і водночас потужних особливостей препроцесорів. Синтаксис оголошення змінної — знак `$` перед ім'ям та її значення після двокрапки.

{% code title="SCSS | main.scss" %}
```scss
$primaryColor: #888;
$cardBorderRadius: 4px;

.product {
  background-color: $primaryColor;
  border-radius: $cardBorderRadius;
}
```
{% endcode %}

{% code title="CSS | main.css" %}
```css
.product {
  background-color: #888;
  border-radius: 4px;
}
```
{% endcode %}

Імена змінних повинні бути описові, щоб за ім'ям було зрозуміло, що там зберігається. Змінна `$color-blue` має деяке значення (крім того, що вона вказує на синій колір), а ось `$color-accent`, `$color-primary` або `$color-background` показують роль цього кольору. Семантичні, описові імена змінних не вимагають перейменування, наприклад, у разі зміни палітри брендових кольорів компанії.

{% hint style="info" %}
Імена змінних можуть бути записані в `kebab-case`, `snake_case` або `camelCase` нотаціях. Головне — щоб у проєкті використовувався тільки один з цих стилів для однорідності коду.
{% endhint %}

## Область видимості

Змінні доступні тільки в межах того рівня вкладеності селекторів, на якому вони визначені. Тобто, якщо змінна оголошена в селекторі, вона доступна тільки в ньому. Змінна, яка оголошується за межами будь-яких селекторів, доступна глобально.

{% code title="SCSS | main.scss" %}
```scss
$primaryColor: red;

body {
  $primaryColor: blue;
  // Це зовсім інша змінна, хоча і з таким самим ім'ям,
  // і доступна вона тільки всередині body.
  background: $primaryColor;
}

p {
  color: $primaryColor;
  // Колір тексту параграфа буде червоний,
  // використовується глобальна змінна зі значенням red.
}
```
{% endcode %}

У разі, якщо в імені змінної допущена помилка або такої змінної немає в доступній області видимості, виникне помилка компіляції стилів.

{% code title="SCSS | main.scss" %}
```scss
$primaryColor: red;
body {
  $secondaryColor: blue;
  background: $primaryColor;
}

p {
  color: $primaryColor;
  background-color: $secondaryColor;
  // Виникне помилка компіляції стилів, тому що
  // змінна secondaryColor не доступна глобально.
  // Вона існує тільки в селекторі body.
}
```
{% endcode %}

{% hint style="warning" %}
Якщо сталася помилка компіляції, необхідно її виправити — після цього виконається повторна компіляція і оновиться результуючий CSS-файл.
{% endhint %}
