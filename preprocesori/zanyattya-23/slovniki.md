# Словники

Оголосимо кілька змінних для зберігання палітри кольорів проекту. Вони слабо пов'язані, по суті - це просто набір змінних з префіксом `color` для зазначення того, що вони зберігають колір.

```scss
// _colors.scss

$color-primary: #8e3329;
$color-accent: #d98328;
$color-secondary: #5a1321;
$color-foreground: #191919;
$color-background: #e9e9e9;
```

Мапа (словник) дозволяє зберігати набір пов'язаної інформації у форматі `ключ: значення`. Наприклад, замість декількох непов'язаних змінних, що зберігають палітру проекту, можна записати мапу кольорів.

`map` - це тип даних в препроцесорі Sass, який постачається у модулі [`sass:map`](https://sass-lang.com/documentation/modules/map/) разом з набором корисних функцій для обробки таких словників.

Перетворимо набір змінних в мапу. Запис мапи буде складатися з імені змінної, в якій зберігається сама мапа, і набору властивостей у форматі `ключ: значення`, розділених комами всередині пари круглих дужок.

```scss
// _colors.scss

$colors: (
  "primary": #8e3329,
  "accent": #d98328,
  "secondary": #5a1321,
  "foreground": #191919,
  "background": #e9e9e9,
);
```

Ім'я кольору (ключ) може бути довільне - це просто назва властивості. Така мапа дозволить впорядковано і багаторазово отримувати доступ до кольорів. Для того щоб отримати значення властивості, використовується функція `map.get($map, $key)` із модуля `sass:map`.

Першим аргументом передаємо ім'я змінної, в якій знаходиться словник (`$map`), другим - ім'я властивості (`$key`), значення якого необхідно отримати. Ім'я властивості повинно бути рядком, тобто взяте в одинарні або подвійні лапки.

```scss
// _colors.scss
@use "sass:map"; // імпортуємо модуль для роботи з мапою

$colors: (
  "primary": #8e3329,
  "accent": #d98328,
  "secondary": #5a1321,
  "foreground": #191919,
  "background": #e9e9e9,
);

body {
  background-color: map.get($colors, "background");
}
```

{% hint style="info" %}
Функція повертає значення, яке є в `$map`, пов’язане з `$key`, якщо ж `$map` не має значення, пов’язаного з `$key`, повертає значення `null`.
{% endhint %}

Постійно писати `map.get()` і передавати ім'я мапи буває незручно. Напишемо свою функцію-утиліту для простішого доступу до властивостей мапи кольорів. Цю функцію оголосимо у тому ж файлі, де знаходиться мапа.

```scss
// _colors.scss
@use "sass:map"; // імпортуємо модуль для роботи з мапою

$colors: (
  "primary": #8e3329,
  "accent": #d98328,
  "secondary": #5a1321,
  "foreground": #191919,
  "background": #e9e9e9,
);

@function get-сolor($key) {
  @return map.get($colors, $key);
}
```

Тепер, щоб встановити значення кольору потрібно імпортувати файл `_colors.scss` і застосувати функцію `get-сolor($color-name)`.

```scss
// _btns.scss
@use 'шлях/до/colors' as c; // імпортуємо модуль для роботи з кольорами

.button {
  color: c.get-color('primary');
  background-color: c.get-color('background');
}
```

Ще один розповсюджений спосіб роботи із мапами - це ітерація по мапі за допогою `@each`

```scss
// _alerts.scss

$types: (
  "primary": #F07F2E,
  "success": #82C43C,
  "danger": #FC5A5A,
  "warning": #A461D8,
  "info": #4E75FF,
);

@each $key, $value in $types {
  .alert-#{$key} {
    background-color: $value;
  }
}
```

Директива `@each` оцінює блок стилів для кожної пари `ключ: значення` мапи. Ключ і значення призначаються змінним, тому до них можна легко отримати доступ у блоці.

За допомогою мапи з набором `ключів` у вигляді типів повідомлень та `значень` у вигляді кольорів ми ефективно згенерували набір класів для повідомлень на сайті.
